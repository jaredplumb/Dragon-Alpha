# Dragon Alpha Gameplay Guide

Last updated: 2026-03-23.

## Purpose
- Define durable gameplay authority for first public macOS release quality.
- Keep loop/progression intent stable as implementation details evolve.

## Core Loop
1. Traverse world map and trigger scripted interactions.
2. Resolve battle/shop/train/status decisions.
3. Save progress and reopen new routes/content.

## First-Push Gameplay Priorities
- World-map interaction reliability over adding new event types.
- Battle readability and deterministic outcome consistency over feature expansion.
- Save/load correctness and progression persistence over content growth.

## Progression Rules
- Player growth must remain legible: level, gear, and map unlock changes are visible and verifiable.
- Event outcomes (talk/treasure/heal/train/shop/gate/challenge/warp) must be reachable through normal map input.
- Deterministic validation should cover gameplay-critical paths before manual-only polish work.

## Economy And Combat Expectations
- Shop pricing/gold deltas must be predictable and regression-tested.
- Battle command outcomes must provide explicit readable feedback (damage, no-damage, absorb/heal cases).
- Multi-enemy continuation and retreat/defeat/victory panels must remain stable across seeded cases.

## Non-Goals For First Push
- Large new gameplay systems not present in the current recovered content.
- Platform expansion beyond macOS.
