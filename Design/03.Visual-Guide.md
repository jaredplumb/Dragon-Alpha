# Dragon Alpha Visual Guide

Last updated: 2026-03-23.

## Purpose
- Define visual/layout authority for first public macOS release readiness.
- Keep scene readability and interaction clarity consistent across regressions.

## Visual Priorities
- Preserve recognizable legacy identity where practical.
- Prioritize readable text and clear state feedback over ornamental changes.
- Keep controls discoverable and touch/click targets reliable.

## Scene Expectations
- Splash/New Game/New Avatar: no clipped labels, clear affordances, stable selection state.
- World Map: map frame, objective line, message bar, and side controls remain within safe bounds.
- Battle: command rows, popups, and result panels stay centered/readable in preferred view.
- Status/Shop: row text, detail lines, and action buttons remain legible at runtime sizes.

## Asset Rules
- Runtime assets remain source-traceable through `Assets/Docs/RuntimeAssetSourceMap.latest.tsv`.
- Avoid fallback placeholders for reachable scenes.
- Maintain alpha/chroma correctness for active runtime sprites and UI elements.

## Validation Contract
- Deterministic scene validation inventory is tracked in `Design/Testing.md`.
- Active visual regression reports live in `Agents/Reports/` (not in `Design/`).
- Native-only pixel-feel signoff is tracked in `Design/Release-Readiness.md`.
